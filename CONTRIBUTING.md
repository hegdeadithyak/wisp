# Contributing to Wisp

First off, thank you for considering contributing to Wisp! We really need help, and with your contributions, this project can be 100 times better than it is right now.

## Pull Request Guidelines

To maintain the quality of the project and make the review process as smooth as possible, please follow these guidelines when submitting a Pull Request (PR):

1. **Keep PRs Neat**: 
   - Ensure your code is clean and well-commented.
   - Keep your commits focused and logical. Avoid giant, monolithic commits that touch unrelated files.
   - Provide a clear and descriptive title and description for your PR.

2. **Attach Proofs**:
   - **Visual Proofs:** For UI/VS Code Extension changes, please attach screenshots or screen recordings showcasing the before and after.
   - **Technical Proofs:** For backend, context-trimming, or performance improvements, please provide benchmarks, logs, or other proofs that demonstrate the improvement.
   - Explain *how* you tested your changes in the PR description.

We welcome all kinds of contributions, ranging from bug fixes, feature additions, and performance tweaks to documentation improvements. Don't hesitate to open an issue or submit a PR!
